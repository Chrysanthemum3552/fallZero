[증상]
- #1을 정면으로 전환한 뒤, 사용자가 가만히 있어도(특히 "1회 수행 → 정지" 시)
  진행도 막대기가 스스로 차올랐다가 ~수초에 걸쳐 다시 0으로 내려오는 현상이 반복됨.

[측정 방식 배경]
- 정면 무릎 펴기 신호:  sig = (무릎Y − 발목Y) / SBU
   · 발이 바닥(굽힘) ≈ −1.2,  다리 폄(발목이 무릎 높이로) ≈ −0.55
- signal = max(0, sig − baseline)      ← "바닥(baseline) 대비 발목이 올라온 정도"
   · 이 signal을 막대기 진행도와 카운트 판정에 사용.

[진단 과정 — 전부 logcat 실측 기반]
 1차) 가설: baseline이 sig의 느린 drift를 못 따라가 누적(creep).
       조치: floor 윈도우 단축(60→40) + 상세 로그. → 부분적, 미해결.

 2차) "1회 후 정지 → 막대기 올랐다 내려옴" 재현 로그에서 발견:
       - baseline을 "윈도우 최저 percentile(10th)"로 계산하고 있었음.
       - rep 직후 MediaPipe 발목 좌표가 순간적으로 바닥보다 더 아래로 튐(하향 글리치, sig ≈ −1.4).
       - 이 낮은 값이 윈도우에 들어가 baseline을 끌어내림 → signal = sig − baseline 이 부풀어 막대기 차오름.
       - 그 글리치가 윈도우(약 4초)에서 빠져나가면 baseline 복귀 → 막대기 0으로 내려옴.
         (= 사용자가 본 "올랐다 슬슬 내려옴"의 정체)
       조치: baseline을 "하향 글리치에 강건한 rate-limited 추적기"로 교체. → 그래도 미해결.

 3차) 새 로그에서 결정적 사실(진짜 원인):
       - 잠금(측정 중)인 다리의 "원시 sig 자체가 휴식 중 −1.0 ↔ −1.3 으로 약 0.3 출렁임"(대칭 jitter).
         (비잠금 다리는 안정 → 측정 대상 다리 좌표가 더 불안정)
       - signal = max(0, sig − floor) 의 "정류(rectify)"로 jitter의 위쪽 절반만 양수로 남음.
       - floor를 어디에 두든, 정류된 신호의 평균이 양수가 되어 스무딩된 막대기가 위로 떠오름.

[최종 원인]
- 핵심: "상대 신호 + max(0,·) 정류" 구조 + 휴식 중 원시 sig의 큰 jitter.
   · 휴식 jitter(±0.15 수준)가 정류되면 위쪽 절반(평균 ~ +0.07)이 그대로 남아 막대기를 채움.
   · (초기 증폭 요인) 윈도우 최저-percentile baseline은 하향 좌표 글리치에 끌려내려가 현상을 더 키움.

[해결]  KneeExtensionEngine.kt 의 다리별 baseline/신호 계산기(FloorTracker)를 재설계
 1) sig EMA 스무딩 (SIG_ALPHA = 0.3)            → jitter 진폭 자체를 축소.
 2) floor = "휴식 평균"을 EMA로 추적 (FLOOR_ALPHA = 0.08),
    단 상승폭이 REST_BAND(0.12)를 넘으면(=다리 폄) floor를 "동결".
      → 휴식 중엔 floor가 sig 평균에 수렴 ⇒ signal = (sigEma − floor) ≈ 0  (정류 bias 제거)
      → 하향 글리치는 sig가 floor 아래라 signal = 0  (글리치 무해화)
      → 다리 폄은 "동결된 floor" 대비 그대로 측정 ⇒ 정상 카운트 유지.
 3) (부수) 막대기 시각 범위를 카운트 임계값과 분리 — 휴식 노이즈 바로 위(0.04)부터 차오르게 해
    "바닥에서 조금만 올려도 반응 + 휴식 땐 빈 막대기".

[결과]
- 가만히 있을 때 막대기 안정(스스로 차오름 없음), 정상 폄은 그대로 잘 카운트. 사용자 확인 완료(2026-06-04).

[교훈 / 재발 방지]
- "상대 신호 + max(0,·) 정류" 구조는 휴식 jitter를 그대로 양수로 흘려보낼 수 있다.
  → (a) 입력(sig) 스무딩, (b) "휴식 중에만 추적하고 동작 중엔 동결"하는 baseline 조합이
     정류 bias를 없애는 정석. 동일 패턴의 다른 정면 운동에도 적용 가능.
- 최저값/저-percentile baseline은 하향 좌표 글리치에 취약하므로, 글리치가 baseline을
  오염시키지 못하는 구조(동결/스무딩)가 더 안전하다.

   · getGuide(): 위 3) 막대기 시각 범위 분리(visFloor 0.04)
